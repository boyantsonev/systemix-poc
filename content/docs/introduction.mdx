A hypothesis validation loop for builders. PostHog measures what you ship. Hermes reads the result and writes it back to the hypothesis contract. The next agent, sprint, or experiment starts from evidence — not a fresh guess.

---

## The loop

1. **You write a hypothesis.** A landing variant, copy test, or onboarding flow — recorded in an MDX contract with the ICP it targets, the claim it makes, and what success looks like.

2. **PostHog measures it.** Feature flags or A/B tests serve your variants. PostHog tracks what matters — conversions, activation, funnel drop-off.

3. **Hermes synthesizes the result.** A local LLM (Ollama, any model) reads the PostHog data against the contract history: prior experiments, rejected directions, original intent.

4. **A decision surfaces.** Hermes queues a HITL card — promote the variant, run longer, or kill it. One click. No forms.

5. **Evidence is written back.** The contract records the decision, the data, and the date. The next agent or experiment starts from this known ground — not a fresh guess.

## The tools it connects

Systemix doesn't replace your stack. It wraps around what you already have.

- **PostHog** — Where your experiment data lives. Systemix reads it — nothing moves.
- **Ollama / Hermes** — Local LLM that synthesizes results. No API key. No cloud. Any Ollama-compatible model works.
- **MDX contracts** — One file per hypothesis in your repo. Machine-readable frontmatter + human-readable rationale. Committed to git alongside your code.
- **MCP server** — Exposes contracts to Claude Code and Cursor. When an AI agent touches something you've already tested, it gets the prior decision — not a guess.
- **Figma** — Optional. Systemix can read Figma for visual context, but you work in your editor — not in a plugin.
- **Vercel** — Deploy target. The post-deploy hook triggers Hermes to pull fresh PostHog data into the contract.

## What it's not

- Not a documentation site — the contract is in your repo, not behind a hosted UI.
- Not a Figma plugin — it reads Figma, but you work in your editor.
- Not a replacement for Storybook — it sits alongside it. Storybook shows what exists. Systemix shows what worked.
- Not a replacement for PostHog — it closes the loop PostHog opens.
- Not a design system tool specifically — for any founder running experiments with AI agents.
- Not locked to Hermes — any Ollama model reads and writes the contracts.
