---
title: What is Systemix?
---

Systemix is a composable hypothesis‚Üívalidate‚Üílearn loop. You write an experiment ‚Äî an ICP, a hypothesis, and the AI workflow that tests it ‚Äî measure it against real signal, and close it with a decision. The learning is written back to `experiments/LEARNINGS.md`, where the next agent or sprint starts from evidence, not a fresh guess ‚Äî because each session starts stateless, the files are its memory.

**Three ways to drive the loop:**
- [Claude Code skills](/docs/slash-commands) ‚Äî `/systemix-init`, `/measure`, `/close-experiment` and more
- CLI ‚Äî `systemix experiment new | list | close | learnings`
- MCP ‚Äî `experiment_new`, `experiment_list`, `experiment_close` for any agent that speaks MCP

The design system is optional. You bring your own, scaffold one, or skip it entirely.
