# Instance model

One Systemix instance per engagement — not per repo, not per app. The entire value of Systemix is a single unified context loop where marketing, business, design, and engineering read from the same evidence base.

## One instance, three surfaces

A Systemix instance exposes three surfaces inside your project:

**Home** — `/config`

View and edit `systemix.config.yaml`. Defines which surfaces are active, which signals are wired in (PostHog, Figma, Vercel), and the autonomy tier (ghost / assisted / autonomous). Also hosts the HITL queue and runtime feed.

**Contract** — `/contract`

The canonical record your agents read from. Goals decompose into hypotheses; hypotheses close with PostHog evidence; memory accumulates per decision. Every write carries provenance. Hermes proposes — you approve. Ghost tier at init: nothing writes autonomously until you raise the trust tier.

**Atlas** — `/atlas`

Agentic workflow catalog organized by persona. Each persona tab shows the workflows available to that role. Clicking a step opens its prototype inline. Atlas is the navigation layer on top of prototypes; the prototype viewer is the detail pane, not a separate surface.

## Config topology

```yaml
# systemix.config.yaml — root of a monorepo engagement
surfaces:
  - design-system
  - landing
  - onboarding

signals:
  posthog:
    enabled: true
    poll_interval_sec: 300
  vercel:
    enabled: true
  figma:
    enabled: true

hermes:
  autonomy: ghost   # ghost | assisted | autonomous
  thresholds:
    high: 0.85
    medium: 0.55

trust:
  hermes_tier: 0   # 0=ghost · 1=assisted · 2=autonomous
```

`surfaces` determines which Hermes loops are active. A landing A/B test and a DS token drift both write to `contract/` and surface in the same HITL queue — under different sections, but one queue.

**Autonomy levels:**

| Tier | Label | Behaviour |
|------|-------|-----------|
| 0 | ghost | Proposes every action — nothing writes without approval |
| 1 | assisted | Writes low-risk records directly; proposes goals, briefs, hypotheses |
| 2 | autonomous | Writes most artifacts automatically; still proposes goals and briefs per covenant |

Goals and briefs are never auto-written at any tier — that is the covenant.

## Contract directory

```
engagement-root/
├── systemix.config.yaml
├── contract/
│   ├── index.mdx             ← root: brief + goals index + memory
│   ├── goals/*.mdx           ← goal contracts (landing-validation, etc.)
│   ├── hypotheses/*.mdx      ← A/B experiments, each linked to a goal
│   ├── tokens/*.mdx          ← token drift records
│   ├── components/*.mdx      ← component parity records
│   └── meta/                 ← self-improvement contracts (when enabled)
└── .systemix/
    ├── queue.json            ← HITL queue, one per instance
    └── systemix.json         ← Figma file key + variable IDs
```

All surfaces share one `contract/` directory and one HITL queue. A DS token drift and a landing page experiment land in the same queue — whoever is on HITL duty sees the full picture.

## ADR-009 superseded

Early design (ADR-009) proposed separate Systemix instances per consuming app. That was wrong. Splitting instances fragments the evidence base — a DS token change that affects both the landing page and the product UI would only register in whichever instance happened to catch it first.

One instance. One context loop. One queue.
