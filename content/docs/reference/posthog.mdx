# PostHog integration

Systemix wires PostHog events into the evidence layer. When a hypothesis is running, Hermes polls the configured PostHog project, caches the relevant event counts and funnel data in the hypothesis contract, and computes a confidence score. When confidence crosses the threshold defined in the contract, Hermes queues a HITL card to close the experiment.

## Setup

Add PostHog to your signals in `systemix.config.yaml`:

```yaml
signals:
  - posthog-eu      # or posthog-us for US Cloud
```

Set your credentials in `.env.local`:

```sh
POSTHOG_HOST=https://eu.posthog.com     # or https://app.posthog.com
POSTHOG_API_KEY=phc_...
POSTHOG_PROJECT_ID=12345
```

## How evidence is collected

Each hypothesis contract declares which PostHog events to track:

```yaml
---
title: "Hero CTA copy — variant B"
status: running
posthog:
  events:
    - name: cta_clicked
      filter: { properties: { variant: "B" } }
    - name: signup_completed
      filter: { properties: { source: "landing_hero" } }
  funnel:
    - cta_clicked
    - signup_completed
  confidence_threshold: 0.95
---
```

Hermes fetches these events on each signal poll, appends them to the contract via `contract_write_evidence`, and recomputes the running conversion rate. When the funnel conversion reaches `confidence_threshold` with statistical significance, Hermes queues a HITL card with the recommendation.

## Feature flags

PostHog feature flags gate variant rendering in your components:

```tsx
import { useFeatureFlagVariantKey } from 'posthog-js/react'

const variant = useFeatureFlagVariantKey('hero-cta-copy')
// variant === 'control' | 'B' | undefined
```

The flag key must match the `name` field in your hypothesis contract frontmatter. The variant key must match the `variant` property in the contract's PostHog event filters.

## Evidence format

Each evidence record written by Hermes looks like this in the contract file:

```yaml
evidence:
  - date: "2026-06-06"
    source: posthog
    events:
      cta_clicked: 1204
      signup_completed: 387
    funnel_rate: 0.322
    note: "Day 3 of 7-day target window"
```

Records accumulate over time so you can see the experiment's trajectory, not just its current state.

## Manual evidence fetch

Run the growth audit skill to fetch evidence on demand without waiting for Hermes's next poll cycle:

```sh
/growth-audit
```

This reads all running hypothesis contracts, queries PostHog, updates the evidence records, and prints a decision brief sorted by confidence score.

## Closing an experiment

When evidence is decision-ready, use the close-experiment skill:

```sh
/close-experiment
```

This records the result and confidence in the hypothesis contract and pushes a synthesis HITL card for human review before any variant code is removed.
